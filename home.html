User Interface [UI]

Coming up with the design is the most complicated piece of a project. It has to be usable, follow the company's brand, appealing to the eye, and perfect for all types of screens.

  • Sketching
  • Wireframes
  • Illustrator
  • Photoshop

Responsive Design [RWD]

I love writing HTML and CSS. I work very hard to make sure every feature on a website looks perfect on every browser and in every width - not just for each device. Finding and learning new attributes that fix style problems is very rewarding.

Reduce the width of your screen to see how passionate I am!

Accessibility [A11y]

A11y is always at the top of my mind in every step of the web process. From UX to the CMS chosen, each piece must be accessible. I've used many browser extensions, as well as Siteimprove to test my design and code against the WCAG 2.1 standards.

Content Management Systems [CMS]

I believe a good content management system can change a content editor's life. I've developed websites in:

  • Drupal (7 & 8)
  • WordPress
  • Kentico
  • Sitecore

Documentation

Documentation is one of my least favorite parts of a project, but I know it's one of the most important. Not only for me when I go back into my code, but for developers and content editors who touch the site after me. I always try to over-comment my code, which helps keep it organized and readable. Content editor documentation is even more important because people should be able to add content to the site I build them. Handing off a site I've created without any worry is a really great feeling.
